 1 Are code changes needed?                                                                                                                                                                                                                             

 • Yes. To better capture outbreak dynamics and avoid structural inconsistencies revealed by updated parameter values (notably m_C), I made one focused ecological improvement: replace the independent coral-loss fractions with a predator-density    
   driven, multi-prey functional response that allocates COTS feeding across corals with prey-switching and saturates with coral availability. I also widened the soft bounds for m_C to avoid penalizing the updated literature value.                 

 2 Explanation of changes                                                                                                                                                                                                                               

 • Assessment (high level):                                                                                                                                                                                                                             
    • Fit to data: A residuals file was not provided in the current files, so I assessed structure only. The current predation terms remove a fixed fraction of each coral group independent of per-capita feeding capacity and predator density (except
      via H(C)). This can decouple coral loss from actual COTS density and lead to misfit during outbreak peaks and busts.                                                                                                                              
    • Project context: Episodic booms require a strong positive feedback (high fecundity and feeding when coral abundant) and a strong negative feedback (rapid prey depletion then starvation). The existing model includes some of this through       
      food_avail in COTS growth and starvation mortality, but the predation structure is oversimplified because it doesn’t scale coral loss with COTS density nor enforce total feeding limits across prey.                                             
    • Missing/oversimplified processes:                                                                                                                                                                                                                 
       • Multi-prey allocation and handling-time/feeding-capacity constraint are missing.                                                                                                                                                               
       • Predation should scale with COTS density and be limited by per-capita consumption, not just a fraction of coral cover.                                                                                                                         
       • The updated literature value for m_C = 2.56 would be heavily penalized by the previous soft upper bound of 2.0 in code.                                                                                                                        
 • Chosen single ecological improvement:                                                                                                                                                                                                                
    • Replace coral predation terms with a predator-density driven multi-prey functional response with prey-switching and saturating removal:                                                                                                           
       • Total per-area feeding rate: cons_rate = f_max * H(C) * C_prev (proportion per year).                                                                                                                                                          
       • Allocate consumption between A and S using preference weights (c_attack_A, c_attack_S) and a switching exponent eta_switch >= 1:                                                                                                               
         alloc_A = (c_attack_A * A^eta) / (c_attack_A * A^eta + c_attack_S * S^eta).                                                                                                                                                                    
       • Saturating coral loss (bounded by availability):                                                                                                                                                                                               
         pred_A_prop = A * (1 - exp(-(cons_rate * alloc_A) / (A + eps))).                                                                                                                                                                               
         pred_S_prop = S * (1 - exp(-(cons_rate * alloc_S) / (S + eps))).                                                                                                                                                                               
    • This creates strong, realistic nonlinear feedbacks: when COTS density increases, per-area predation increases but is shared between coral types based on availability and preference, and it cannot exceed what is available. This helps produce  
      boom-bust dynamics.                                                                                                                                                                                                                               
    • I preserved the existing Type-III dependence on COTS density via H(C) to maintain low predation at very low predator densities.                                                                                                                   
    • I widened the soft bound on m_C to avoid penalizing the updated literature value.                                                                                                                                                                 
 • Parameter changes:                                                                                                                                                                                                                                   
    • Added f_max (maximum per-capita coral consumption rate per COTS per year; proportion indiv^-1 year^-1).                                                                                                                                           
    • Added eta_switch (prey-switching exponent; dimensionless).                                                                                                                                                                                        
    • Reinterpreted c_attack_A and c_attack_S as preference weights (dimensionless) rather than year^-1 attack rates; descriptions and units updated in parameters.json. Values retained.                                                               
    • Increased the soft upper bound on m_C in model.cpp from 2.0 to 5.0 to respect updated literature value 2.56.
